## Goal


Apply multilingual support and SEO optimization to a Hugo blog to maximize search engine exposure and serve the appropriate language version to multilingual users.


---


## SEO Configuration


SEO (Search Engine Optimization) is the process of optimizing site structure and metadata so that search engines like Google can better understand and surface your content in search results.


This document summarizes the SEO settings applied to the blog.


### 1. Absolute URL - Hugo baseURL Setting

- **File**: `hugo/hugo.toml`
- **Content**: `baseURL = '`[`https://blog.plzhans.com`](https://blog.plzhans.com/)`'`
- Generates correct absolute URLs in sitemap.xml, RSS feeds, Open Graph, etc.
- Sitemap (`sitemap.xml`) and RSS feed (`index.xml`) are auto-generated by Hugo
- `hugo server` (development) automatically uses [`localhost:1313`](http://localhost:1313/), so no separate handling is needed

### 2. Automatic robots.txt Generation

- **File**: `hugo/hugo.toml`
- **Content**: `enableRobotsTXT = true`
- Automatically generates `robots.txt` during Hugo build (allows all crawlers + includes Sitemap URL)

### 3. [Schema.org](http://schema.org/) Structured Data (JSON-LD)

- **File**: `hugo/layouts/_default/single.html`
- Inserts `BlogPosting` JSON-LD on article pages (`type != "page"`)
- Included fields: headline, datePublished, dateModified, author, description, mainEntityOfPage
- Enables rich snippets (author, date, etc.) in Google search results

### 4. og:image (Featured Image) / Open Graph

- **File**: `src/services/NotionExportService.mjs`
- During Notion sync, detects the first image in the content and automatically adds it to the front matter `images` field
- Open Graph meta tags are rendered by Hugo's built-in template (`_internal/opengraph.html`), using `images` as `og:image`

### 5. meta description / Twitter Card

- **File**: `src/services/NotionExportService.mjs`
- Outputs Notion's "Summary" property to the front matter `description` field
- Used in Hugo's built-in opengraph/twitter_cards templates and baseof.html's meta description
- Twitter Card meta tags are rendered by Hugo's built-in template (`_internal/twitter_cards.html`)
- Other meta tags (author, viewport) are provided by the theme by default

### 6. Canonical URL

- **File**: `hugo/layouts/_default/baseof.html`
- Overrides the theme (`m10c`)'s `baseof.html` to add the `<link rel="canonical">` tag
- Uses `.Permalink` as the canonical URL
- Also includes multilingual hreflang tags (outputs `alternate` + `x-default` when translation pages exist)

### 7. Google Analytics (GA4)

- Provided by default in the theme (`m10c`)
- Can be verified through GA integration during Google Search Console authentication

## Key Elements of Multilingual SEO


### HTML lang Attribute


Specifies the page language to provide language information to search engines and screen readers.


```html
<html lang="ko">
```


### link rel alternate hreflang


Informs search engines of the URL for each language version to prevent duplicate content issues.


```html
<link rel="alternate" hreflang="ko" href="https://blog.plzhans.com/ko/post/example/">
<link rel="alternate" hreflang="en" href="https://blog.plzhans.com/en/post/example/">
<link rel="alternate" hreflang="ja" href="https://blog.plzhans.com/ja/post/example/">
<link rel="alternate" hreflang="x-default" href="https://blog.plzhans.com/ko/post/example/">
```


### Canonical URL (Multilingual)


If each language version is written as a professional translation, the canonical can be omitted to have all language versions recognized as independent originals.


```html
<link rel="canonical" href="https://blog.plzhans.com/ko/post/example/">
```

## Hugo Multilingual Implementation


### 1. Verify Theme Multilingual Support


**Check lang attribute** (`themes/{theme}/layouts/_default/baseof.html`)


```html
<!doctype html>
<html lang=" .Site.Language.Lang ">
```


**Check relLangURL support**


Verify that home links maintain language-specific URLs. If not supported, override baseof.html.


```html
<body>
  <header class="app-header">
    <a href=" .Site.Home.RelPermalink "><img class="app-header-avatar" src="..." alt="..." /></a>
```


### 2. hugo.toml Multilingual Configuration


```toml
# Default content language
defaultContentLanguage = "ko"
# Include default language in subdirectory (/ko/)
defaultContentLanguageInSubdir = true

[languages]
  [languages.ko]
    weight = 1
    languageName = "한국어"

  [languages.en]
    weight = 2
    languageName = "English"

  [languages.ja]
    weight = 3
    languageName = "日本語"
```


### 3. Add Canonical Tag


If the theme does not support it, override baseof.html.


```html
<link rel="canonical" href=" .Permalink " />
```


### 4. Generate hreflang Tags


**Set translationKey in content files**


```markdown
---
id: "80"
translationKey: "80"
slug: "80-redis-dump-vs-aof"
title: "Redis dump vs aof"
---
```


**Add hreflang to baseof.html** (override if theme does not support it)


```html
<link rel="alternate" hreflang=" .Language.Lang " href=" .Permalink " />
<link rel="alternate" hreflang="x-default" href=" .Permalink " />
```

## Troubleshooting


### URL Duplication Conflict


When setting post URLs in Hugo, you should use `slug` instead of `url`.


**Cause**

- Setting `slug` automatically adds language prefixes like `/ko/`, `/en/`
- Force-setting `url` prevents Hugo from automatically adding the language code
- When using `url`, you must manually include the language code in the url itself, like `/ko/post/example`, `/en/post/example`
- If the same path is specified in `url` without a language code, posts in different languages will have the same URL, causing a conflict

**Solution**

- Switch from `url` to `slug`
- Set `defaultContentLanguageInSubdir = true` in `hugo.toml` so that all languages, including the default, use the subdirectory structure

**Note**

- When only `slug` is specified, the language code is added automatically, but if the slug itself is written in a specific language, it should be translated for each language. It is recommended to write slugs in English.

### hreflang Not Generated After Adding translationKey


**Cause**

- The theme does not support hreflang tag generation

**Solution**

- Override baseof.html to add hreflang-related code
